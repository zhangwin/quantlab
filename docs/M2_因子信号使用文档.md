## 模块简介

M2 因子信号管线是 QuantLab 三条信号管线之一（管线 A），负责：

1. **因子管理** — 维护因子池（Alpha158 基础集 + 自定义因子），YAML 持久化
2. **因子验证** — 评估因子质量（IC、ICIR、多空夏普、冗余检测），自动判定 accept/reject/review
3. **因子挖掘** — 半自动探索新因子（参数搜索、组合生成、全自动挖掘）
4. **信号生产** — LightGBM 滚动训练 + 日频截面预测

所有脚本位于 `quantlab/signal/` 目录下。

**与其他模块的关系：**

```
M1 DataManager ──→ M2 信号 ──→ M5 信号融合
                      ↑
                 M4 RD-Agent（进化出新因子后注册到因子池）
```

## 快速开始

```bash
cd quantlab/signal

# 第一步：初始化因子池（自动生成 158 个 Alpha158 基础因子）
python factor_registry_cli.py init

# 第二步：查看因子池
python factor_registry_cli.py list

# 第三步：验证某个因子的质量
python factor_validate.py --expr "Mean($close, 20)/$close" --start 2021-01-01 --end 2024-12-31

# 第四步：单日预测（自动训练 LightGBM + 输出信号）
python signal_alpha_predict.py --anchor-date 2024-06-28

# 第五步：添加一个自定义因子
python factor_registry_cli.py add --name VOL_SKEW --expr "Skew($volume, 20)" --category liquidity

# 第六步：全自动挖掘因子
python factor_mine.py auto --budget 100 --register
```

## 脚本详细说明

### factor_registry_cli.py 因子池管理

管理因子池的增删改查，所有变更持久化到 `configs/factors.yaml`。

**查看因子：**

```bash
# 查看全部因子
python factor_registry_cli.py list

# 按来源筛选（alpha158 / manual / rdagent / miner）
python factor_registry_cli.py list --source manual

# 按分类筛选（momentum / volatility / liquidity / mean_revert / custom）
python factor_registry_cli.py list --category momentum

# 因子概览表（含 IC、ICIR 等验证结果）
python factor_registry_cli.py summary
```

输出示例：

```
共 158 个因子 (总计 158 个)

Name                 Category       Source     Enabled  Expression
----------------------------------------------------------------------------------------------------
KMID                 momentum       alpha158   Y        ($close-$open)/$open
KLEN                 volatility     alpha158   Y        ($high-$low)/$open
...
```

**添加因子：**

```bash
# 添加自定义因子
python factor_registry_cli.py add \
    --name VOL_SKEW_20 \
    --expr "Skew($volume, 20)" \
    --category liquidity

# 添加 RD-Agent 进化出的因子
python factor_registry_cli.py add \
    --name RDAGENT_001 \
    --expr "Corr(Std($close,5), Mean($volume,10), 20)" \
    --category mean_revert \
    --source rdagent
```

**删除 / 启用 / 禁用：**

```bash
# 删除因子（仅自定义因子可删除，alpha158 因子只能禁用）
python factor_registry_cli.py remove --name VOL_SKEW_20

# 禁用（不参与训练和预测，但保留在因子池中）
python factor_registry_cli.py disable --name RSQR5

# 重新启用
python factor_registry_cli.py enable --name RSQR5
```

**重新初始化：**

```bash
# 覆盖现有因子池，重新生成 158 个 Alpha158 基础因子
python factor_registry_cli.py init
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--config` | `configs/factors.yaml` | 因子配置文件路径 |

### factor_validate.py 因子验证

评估因子质量，输出 IC 均值、ICIR、多空夏普、与现有因子的相关性等指标，并自动给出 accept/reject/review 判定。

**验证单个因子：**

```bash
python factor_validate.py \
    --expr "Mean($close, 20)/$close" \
    --start 2021-01-01 \
    --end 2024-12-31
```

输出示例：

```
验证因子: Mean($close, 20)/$close
区间: 2021-01-01 ~ 2024-12-31
股票池: csi300

==================================================
  判定结果: ACCEPT
==================================================
  IC 均值:        0.0312
  ICIR:           1.45
  IC 正比率:      62.3%
  多空夏普:       2.15
  自相关:         0.952
  换手率:         0.087
  IC 衰减:        {1: 0.0312, 2: 0.0285, 3: 0.0241, 5: 0.0165, 10: 0.0082}

  → 因子质量良好，建议纳入因子池
```

**验证并检查与现有因子池的冗余：**

```bash
python factor_validate.py \
    --expr "Mean($close, 21)/$close" \
    --start 2021-01-01 \
    --end 2024-12-31 \
    --check-corr
```

**横向对比多个因子：**

```bash
python factor_validate.py \
    --compare "Mean($close,5)/$close" "Mean($close,20)/$close" "Mean($close,60)/$close" \
    --start 2021-01-01 \
    --end 2024-12-31
```

输出对比表：

```
expression                                           ic_mean    icir  ic_pos_ratio  sharpe_ls  auto_corr  turnover  verdict
Mean($close,5)/$close                                 0.0180    0.82         0.578       1.35      0.891     0.152  review
Mean($close,20)/$close                                0.0312    1.45         0.623       2.15      0.952     0.087  accept
Mean($close,60)/$close                                0.0275    1.21         0.601       1.88      0.978     0.045  accept
```

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--expr` | 二选一 | — | 单个因子表达式 |
| `--compare` | 二选一 | — | 多个因子表达式（空格分隔） |
| `--start` | 否 | `2020-01-01` | 验证起始日期 |
| `--end` | 否 | `2024-12-31` | 验证截止日期 |
| `--check-corr` | 否 | 否 | 检查与现有因子池的相关性 |
| `--market` | 否 | `csi300` | 股票池 |

**自动判定标准：**

| 判定 | 条件 |
|------|------|
| **accept** | IC > 0.02 且 ICIR > 0.5 且 最大相关 < 0.7 且 IC正比 > 55% |
| **reject** | IC < 0.01 或 ICIR < 0.2 或 最大相关 > 0.9 |
| **review** | 介于 accept 和 reject 之间，需人工判断 |

### factor_mine.py 因子挖掘

四种挖掘模式：参数搜索、组合探索、模板探索、全自动挖掘。

**参数搜索（对一个因子模板搜索最优参数）：**

```bash
# 搜索波动率因子的最优窗口
python factor_mine.py variants \
    --template "Std($close, {window})/$close" \
    --param window=5,10,20,30,60

# 搜索量价相关性的最优窗口和字段
python factor_mine.py variants \
    --template "Corr({field}, Log($volume+1), {w})" \
    --param "field=$close,$high-$low" --param "w=10,20,30,60"
```

**组合探索（字段 × 算子 × 窗口的笛卡尔积）：**

```bash
python factor_mine.py combinations \
    --fields "$close/Ref($close,1),$volume,Log($volume+1)" \
    --operators "Corr,Cov" \
    --windows 10,20,30,60
```

**模板探索（自定义模板 + 变量替换）：**

```bash
# 探索一系列布林带变种因子
python factor_mine.py template \
    --template "({field}-Mean({field},{w}))/(Std({field},{w})+1e-12)" \
    --variables "field=$close,$volume,$high-$low;w=5,10,20,60"
```

**全自动挖掘（生成候选 → 验证 → 去冗余 → 排序）：**

```bash
# 生成 200 个候选因子，验证并排序
python factor_mine.py auto --budget 200

# 挖掘并自动注册通过验证的因子到因子池
python factor_mine.py auto --budget 500 --register
```

输出示例：

```
全自动挖掘: budget=200

Rank  ICIR     IC       Sharpe   Verdict  Expression
------------------------------------------------------------------------------------------
1     1.82     0.0385   2.68     accept   Corr($close/Ref($close,1), Log($volume+1), 20)
2     1.54     0.0321   2.24     accept   Std($high-$low, 30)
3     1.38     0.0298   1.95     accept   Slope($volume, 60)
...

共 12 个因子通过验证（accept）
已注册 12 个新因子到因子池
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--budget` | `200` | 全自动挖掘的候选因子上限 |
| `--register` | 否 | 自动注册 accept 的因子到因子池 |
| `--start` | `2020-01-01` | 验证区间 |
| `--end` | `2024-12-31` | 验证区间 |

## Python API 参考

### FactorRegistry 因子注册表

```python
from quantlab.signal.signal_alpha import FactorRegistry

reg = FactorRegistry("configs/factors.yaml")  # 首次自动生成 Alpha158
```

**增删改查：**

```python
# 添加因子（name 重复会报错）
reg.add("MY_FACTOR", "Corr($close, $volume, 20)", "liquidity", "manual")

# 删除因子（alpha158 来源的不能删除，只能 disable）
reg.remove("MY_FACTOR")

# 启用 / 禁用
reg.disable("RSQR5")
reg.enable("RSQR5")

# 查询
reg.get("KMID")                     # → FactorDef
reg.list(category="momentum")       # → List[FactorDef]
reg.list(source="manual")           # → List[FactorDef]
reg.get_enabled()                   # → 所有 enabled=True 的因子

# 获取表达式（直接传给 Qlib）
expressions, names = reg.get_expressions()

# 概览表
df = reg.summary()                  # → DataFrame

# 保存到 YAML
reg.save()

# 因子池 hash（用于检测变更触发重训）
reg.factor_hash                     # → "a3f8b1..."
```

**因子定义数据结构（FactorDef）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | str | 唯一标识，如 `"VOL_SKEW_20"` |
| `expression` | str | Qlib 表达式，如 `"Skew($volume, 20)"` |
| `category` | str | `momentum` / `mean_revert` / `volatility` / `liquidity` / `custom` |
| `source` | str | `alpha158` / `manual` / `rdagent` / `miner` |
| `enabled` | bool | 是否参与训练和预测 |
| `added_date` | str | 添加日期 |
| `validation` | dict | 最近验证结果（ic_mean, icir 等） |

### FactorValidator 因子验证器

```python
from quantlab.data.data_manager import DataManager
from quantlab.signal.signal_alpha import FactorValidator

dm = DataManager(provider_uri="~/.qlib/qlib_data/cn_data", market="csi300")
dm.init_qlib()
validator = FactorValidator(dm)
```

**验证单个因子：**

```python
report = validator.validate_single(
    "Corr($close, $volume, 20)",
    start="2021-01-01",
    end="2024-12-31",
    existing_expressions=expressions,  # 可选，用于冗余检测
)

print(report.verdict)              # "accept" / "reject" / "review"
print(report.ic_mean)              # 0.035
print(report.icir)                 # 1.2
print(report.ic_positive_ratio)    # 0.62
print(report.sharpe_long_short)    # 2.1
print(report.ic_decay)             # {1: 0.035, 2: 0.031, 3: 0.026, 5: 0.018, 10: 0.009}
print(report.max_corr_with_existing)  # 0.28
```

**批量验证 / 横向对比：**

```python
reports = validator.validate_batch(
    ["Mean($close,5)/$close", "Mean($close,20)/$close", "Std($close,20)/$close"],
    start="2021-01-01", end="2024-12-31",
)

df = validator.compare(
    ["Mean($close,5)/$close", "Mean($close,20)/$close"],
    start="2021-01-01", end="2024-12-31",
)
```

**增量验证（评估新因子相对于现有因子的边际贡献）：**

```python
report = validator.validate_incremental(
    "Corr(Std($close,5), Mean($volume,10), 20)",
    existing_expressions=expressions,
)
print(report.correlation_matrix)  # 与 Top-10 因子的相关性矩阵
```

**验证报告字段（FactorReport）：**

| 字段 | 说明 |
|------|------|
| `ic_mean` | 日频 Rank IC 均值 |
| `icir` | IC 均值 / IC 标准差 |
| `ic_positive_ratio` | IC > 0 的交易日占比 |
| `ic_decay` | IC 在 T+1 ~ T+10 的衰减 |
| `sharpe_long_short` | Top20% 做多 - Bottom20% 做空 的年化夏普 |
| `auto_corr` | 因子自相关性（T 和 T-1 截面相关） |
| `turnover` | Top 组日均换手率 |
| `max_corr_with_existing` | 与现有因子的最大截面相关系数 |
| `verdict` | 自动判定：`accept` / `reject` / `review` |

### FactorMiner 因子挖掘器

```python
from quantlab.signal.signal_alpha import FactorMiner

miner = FactorMiner(validator, registry)
```

**参数搜索：**

```python
reports = miner.explore_variants(
    base_expression="Std($close, {window})/$close",
    param_grid={"window": [5, 10, 20, 30, 60]},
)
# 返回 5 个 FactorReport，按 ICIR 降序
```

**组合探索：**

```python
reports = miner.explore_combinations(
    fields=["$close/Ref($close,1)", "$volume", "Log($volume+1)"],
    operators=["Corr", "Cov"],
    windows=[10, 20, 30, 60],
)
# 生成 2×3×2×4 = 24 个候选（去掉 field1==field2），按 ICIR 降序
```

**模板探索：**

```python
reports = miner.explore_from_template(
    template="({field} - Mean({field}, {window})) / (Std({field}, {window}) + 1e-12)",
    variables={"field": ["$close", "$volume", "$high-$low"], "window": [5, 10, 20, 60]},
)
# 3 × 4 = 12 个布林带变种
```

**全自动挖掘：**

```python
reports = miner.auto_mine(budget=500)
# 搜索空间：9 字段 × 11 算子 × 5 窗口 ≈ 1800+ 候选
# 采样 500 个 → 批量验证 → 去冗余 → 排序返回

# 自动注册通过的因子
count = miner.accept_and_register(reports)
print(f"注册了 {count} 个新因子")
```

### AlphaSignalPipeline 信号生产

```python
from quantlab.signal.signal_alpha import FactorRegistry, AlphaSignalPipeline

reg = FactorRegistry("configs/factors.yaml")
pipeline = AlphaSignalPipeline(
    registry=reg,
    market="csi300",
    retrain_interval=20,  # 每 20 个交易日重训
    train_years=3,         # 训练窗口 3 年
)
```

**预测：**

```python
signal = pipeline.predict("2024-06-28", dm)
# → Series[symbol → score]，score 越高越看多
# 首次调用自动触发训练

# 后续调用不重训（除非到了 retrain_interval 或因子池变更）
signal2 = pipeline.predict("2024-07-01", dm)
```

**特征重要性：**

```python
importance = pipeline.get_feature_importance()
# → Series[factor_name → importance]，gain-based
print(importance.head(10))
```

**自动重训逻辑：**

```python
pipeline.should_retrain("2024-06-28")  # → True/False
# 触发条件（满足任一）：
# 1. 从未训练过
# 2. 累计预测次数 ≥ retrain_interval
# 3. registry.factor_hash 发生变化（添加/删除/启用/禁用了因子）
```

**训练标签说明：**

训练数据的最后一天 = `anchor_date - 2 交易日`，标签 = `Ref($close, -2) / Ref($close, -1) - 1`。这是因为 anchor-2 日的标签需要用到 anchor 日的 close（刚好可用），而 anchor-1 日的标签需要 anchor+1 日的 close（不可用）。

## 因子表达式语法

Qlib 因子是**字符串表达式**，由基础字段和算子组合而成。新增因子 = 写一行表达式，不需要写 Python 代码。

**基础字段：**

| 字段 | 说明 |
|------|------|
| `$open` | 开盘价 |
| `$high` | 最高价 |
| `$low` | 最低价 |
| `$close` | 收盘价 |
| `$volume` | 成交量 |
| `$vwap` | 成交均价 |

**常用算子（50+ 个，以下为最常用）：**

| 算子 | 语法 | 说明 |
|------|------|------|
| `Ref` | `Ref($close, 5)` | 5 天前的值 |
| `Mean` | `Mean($close, 20)` | 20 日均值 |
| `Std` | `Std($close, 20)` | 20 日标准差 |
| `Slope` | `Slope($close, 20)` | 20 日线性回归斜率 |
| `Rsquare` | `Rsquare($close, 20)` | R² 拟合度 |
| `Rank` | `Rank($close, 20)` | 百分位排名 |
| `Max` / `Min` | `Max($high, 20)` | 窗口最大/最小值 |
| `Quantile` | `Quantile($close, 20, 0.8)` | 80% 分位数 |
| `Corr` | `Corr($close, $volume, 20)` | 20 日相关系数 |
| `Skew` / `Kurt` | `Skew($close, 20)` | 偏度/峰度 |
| `Sum` | `Sum($volume, 20)` | 20 日累加 |
| `IdxMax` / `IdxMin` | `IdxMax($high, 20)` | 最大值出现在几天前 |
| `Greater` / `Less` | `Greater($open, $close)` | 取较大/较小值 |
| `Log` | `Log($volume+1)` | 取对数 |
| `Abs` | `Abs($close-$open)` | 绝对值 |
| `Resi` | `Resi($close, 20)` | 线性回归残差 |

**示例表达式：**

```
Mean($close, 5)/$close                            5 日均线偏离
Std($close, 20)/$close                            20 日波动率
Corr($close/Ref($close,1), Log($volume+1), 30)   量价相关性
Slope($close, 10)/$close                          10 日趋势
($close-Min($low,20))/(Max($high,20)-Min($low,20)+1e-12)   RSV 随机指标
```

## Alpha158 因子集说明

Alpha158 是 Qlib 内置的 158 个技术因子集，分三大类：

| 类别 | 因子数 | 说明 |
|------|--------|------|
| **Kbar** | 9 | K 线形态特征（KMID, KLEN, KUP, KLOW, KSFT 等） |
| **Price** | 4 | 归一化价格（OPEN0, HIGH0, LOW0, VWAP0） |
| **Rolling** | 145 | 滚动窗口算子 × 5 个窗口（5/10/20/30/60） |

**Rolling 因子包含 29 类算子（每类 5 个窗口）：**

| 算子 | 因子名 | 含义 |
|------|--------|------|
| ROC | ROC5~60 | 价格变动率 |
| MA | MA5~60 | 均线偏离 |
| STD | STD5~60 | 价格波动率 |
| BETA | BETA5~60 | 线性趋势斜率 |
| RSQR | RSQR5~60 | 趋势拟合度 |
| RESI | RESI5~60 | 趋势偏离度 |
| MAX/MIN | MAX5~60, MIN5~60 | 窗口极值 |
| QTLU/QTLD | QTLU5~60, QTLD5~60 | 80%/20% 分位 |
| RANK | RANK5~60 | 历史百分位 |
| RSV | RSV5~60 | 随机指标 |
| IMAX/IMIN/IMXD | IMAX5~60 等 | Aroon 指标 |
| CORR/CORD | CORR5~60 等 | 量价相关 |
| CNTP/CNTN/CNTD | CNTP5~60 等 | 涨跌天数比 |
| SUMP/SUMN/SUMD | SUMP5~60 等 | RSI 类动量 |
| VMA/VSTD/WVMA | VMA5~60 等 | 成交量特征 |
| VSUMP/VSUMN/VSUMD | VSUMP5~60 等 | 成交量 RSI |

所有因子均做了**去量纲处理**（除以当前 close 或 volume），使不同价格水平的股票可比。

## 常见问题

### Q: 因子池文件在哪？

默认在 `quantlab/configs/factors.yaml`。首次运行会自动生成 158 个 Alpha158 因子。

### Q: 如何让 RD-Agent（M4）进化出的因子加入因子池？

M4 进化出新因子表达式后，调用 registry 注册：

```bash
python factor_registry_cli.py add \
    --name RDAGENT_001 \
    --expr "Corr(Std($close,5), Mean($volume,10), 20)" \
    --category mean_revert \
    --source rdagent
```

或在 Python 中：

```python
registry.add("RDAGENT_001", evolved_expression, "mean_revert", "rdagent")
```

注册后下次 `predict()` 会自动检测到因子池变更并触发重训。

### Q: 训练很慢怎么办？

- 缩短训练窗口：`--train-years 2`
- 减少因子数：禁用不需要的因子类
- 调整 LightGBM 参数：减少 `num_boost_round` 或 `num_leaves`

### Q: 为什么训练标签用 `Ref($close, -2)/Ref($close, -1) - 1` 而不是简单的次日收益？

这是 T+1 到 T+2 的收益，而不是 T 到 T+1。原因是 A 股 T+1 交易制度下，模型预测的信号在 T+1 日开盘执行，关心的是 T+1 买入后到 T+2 的收益。与 Qlib 官方 Alpha158 benchmark 的标签定义一致。

### Q: 自动挖掘 budget 设多少合适？

- 快速试探：100~200（几分钟）
- 正式挖掘：500~1000（十几分钟到几十分钟）
- 大规模搜索：2000+（需要较长时间）

budget 控制的是从搜索空间中采样的候选数量，不是最终注册的因子数。

### Q: 如何只用部分因子训练模型？

禁用不需要的因子，只保留想要的：

```bash
# 禁用所有流动性因子
python factor_registry_cli.py list --category liquidity
# 逐个禁用，或在 Python 中批量操作：
```

```python
for fd in reg.list(category="liquidity"):
    reg.disable(fd.name)
reg.save()
```
