## 模块简介

M5 信号融合是 QuantLab 的信号汇聚层，负责将三条管线的信号合成为统一的选股信号：

1. **Horizon 对齐** — 将各管线不同频率/值域的信号统一到日频 rank 空间
2. **动态加权** — 基于滚动 Rank IC 自适应调整各管线权重，含冷启动、权重钳制、相关性感知
3. **信号质量监控** — 跟踪各管线贡献度变化，检测信号退化

所有脚本位于 `quantlab/signal/` 目录下。

**在系统中的位置：**

```
M2 Alpha158 ─→ alpha 信号 ──┐
                              │
M3 Kronos ───→ kronos 信号 ──┼──→ M5 信号融合 ──→ 融合信号 ──→ M6 交易执行
               uncertainty ──┤
                              │
M4 RD-Agent ─→ rdagent 信号 ─┘
```

**三条管线的信号特性差异：**

| | M2 Alpha158 | M3 Kronos | M4 RD-Agent |
|---|---|---|---|
| 值域 | 不固定（模型原始输出） | 收益率预测（通常 ±3%） | 不固定（因子加权值） |
| 分布 | 连续，可能有极端值 | 偏态，较稳定 | 连续，可能有极端值 |
| 额外信息 | 无 | uncertainty（不确定性） | 可能为空（无 active 因子） |
| 缩尾需求 | 是（1%/99%） | 否（分布已稳定） | 是（1%/99%） |

**融合解决的问题：**

| 问题 | 解决方案 |
|------|---------|
| 量纲差异 | Rank 归一化到 (0, 1] |
| 权重分配 | 滚动 IC 加权（非固定权重） |
| 冗余信号 | 相关性感知降权 |
| 预测不确定 | Kronos uncertainty 个股级惩罚 |
| 冷启动 | 等权 → 混合 → IC 加权，渐进过渡 |
| 管线缺失 | 优雅降级（C 管线可空） |

## 环境准备

M5 依赖极少，M1 环境已装好即可：

```bash
pip install numpy pandas scipy   # 通常已安装
```

无需 GPU、无需 Qlib 数据（M5 本身不访问原始数据，只处理上游管线输出的信号）。

## 快速开始

### 从 CSV 文件融合

```bash
cd quantlab/signal

# 假设各管线信号已保存为 CSV
# alpha_signals.csv:  行=日期, 列=股票, 值=信号得分
# kronos_signals.csv: 同上
# rdagent_signals.csv: 同上

# 第一步：单日融合
python ensemble_combine.py --anchor-date 2024-06-28 \
    --alpha-signal ./outputs/alpha/signals.csv \
    --kronos-signal ./outputs/kronos/signals.csv \
    --rdagent-signal ./outputs/rdagent/signals.csv

# 第二步：批量融合
python ensemble_combine.py --start 2024-01-01 --end 2024-06-30 \
    --alpha-signal ./outputs/alpha/signals.csv \
    --kronos-signal ./outputs/kronos/signals.csv \
    --output ./outputs/ensemble/signals.csv

# 第三步：回测监控（需要实际收益 CSV）
python ensemble_monitor.py backtest \
    --alpha-signal ./outputs/alpha/signals.csv \
    --kronos-signal ./outputs/kronos/signals.csv \
    --return-csv ./outputs/returns.csv \
    --start 2024-01-01 --end 2024-06-30

# 第四步：查看 IC 历史
python ensemble_monitor.py ic-history \
    --alpha-signal ./outputs/alpha/signals.csv \
    --kronos-signal ./outputs/kronos/signals.csv \
    --return-csv ./outputs/returns.csv

# 第五步：告警检查
python ensemble_monitor.py check \
    --signal-csv ./outputs/ensemble/signals.csv \
    --return-csv ./outputs/returns.csv
```

### Python API 融合

```python
from quantlab.signals.signal_ensemble import SignalEnsemblePipeline

ensemble = SignalEnsemblePipeline(
    ic_lookback=60,
    uncertainty_penalty=0.1,
    min_weight=0.1,
    max_weight=0.6,
)

# 接收各管线信号
signals = {
    "alpha": alpha_pipeline.predict(date, dm),           # M2 输出
    "kronos": kronos_output.return_1d,                    # M3 输出
    "kronos_uncertainty": kronos_output.uncertainty,      # M3 不确定性
    "rdagent": rdagent_output.signal,                     # M4 输出（可能为空）
}

# 融合
output = ensemble.combine(signals, "2024-06-28")
print(output.signal)    # → M6 选股
print(output.weights)   # 各管线权重
print(output.mode)      # "cold_start" / "blending" / "ic_weighted"

# 次日回填实际收益（用于 IC 计算）
ensemble.update_history("2024-06-28", signals, actual_return)
```

## Python API 参考

### SignalNormalizer 信号预处理

将各管线原始信号统一到可比较的尺度。全部为静态方法。

```python
from quantlab.signals.signal_ensemble import SignalNormalizer

# Rank 归一化
signal = pd.Series({"A": 10, "B": 30, "C": 20, "D": np.nan})
ranked = SignalNormalizer.rank_normalize(signal)
# A: 0.333, B: 1.0, C: 0.667, D: NaN

# 缩尾处理
extreme = pd.Series(range(1, 101))
winsorized = SignalNormalizer.winsorize(extreme, lower=0.05, upper=0.95)
# min ≈ 5, max ≈ 95

# Index 对齐
s_a = pd.Series([1, 2, 3], index=["A", "B", "C"])
s_b = pd.Series([4, 5, 6], index=["B", "C", "D"])
aligned = SignalNormalizer.align_index({"a": s_a, "b": s_b})
# 两者都有 index ["A", "B", "C", "D"]，缺失为 NaN
```

**处理顺序：**

```
原始信号 → winsorize（去极值，1%/99%） → rank_normalize（归一化到 (0, 1]）
```

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `rank_normalize(signal)` | Series | Series | 截面 rank 归一化到 (0, 1]，NaN 保持 |
| `winsorize(signal, lower, upper)` | Series, 分位数 | Series | 缩尾处理（默认 1%/99%） |
| `align_index(signals)` | Dict[name, Series] | Dict[name, Series] | 对齐股票 index，缺失填 NaN |

**缩尾时机：**

- alpha 和 rdagent 管线：先缩尾再 rank（防极端离群值扭曲排名）
- kronos 管线：不缩尾（输出为收益率预测，分布已稳定）

### ICWeightEngine IC 动态加权引擎

基于各管线历史预测准确度动态分配权重。核心是滚动 Rank IC。

```python
from quantlab.signals.signal_ensemble import ICWeightEngine

engine = ICWeightEngine(
    ic_lookback=60,           # IC 窗口长度（天）
    min_weight=0.1,           # 单管线最低权重
    max_weight=0.6,           # 单管线最高权重
    correlation_penalty=0.3,  # 相关性惩罚系数
)

# 逐日追加历史（T-1 信号 vs T 收益）
engine.update("2024-06-27", signals_dict, actual_return_series)

# 计算当前权重
weights, mode = engine.compute_weights()
# weights: {"alpha": 0.45, "kronos": 0.33, "rdagent": 0.22}
# mode: "cold_start" | "blending" | "ic_weighted"

# 查看某管线 IC
ic = engine.compute_ic("alpha")   # 近 60 天平均 Rank IC

# 获取逐日 IC 历史
ic_df = engine.get_ic_history()   # DataFrame: date × pipeline_name
```

**构造参数：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `ic_lookback` | int | `60` | IC 滚动窗口长度（天） |
| `min_weight` | float | `0.1` | 单管线最低权重（10%） |
| `max_weight` | float | `0.6` | 单管线最高权重（60%） |
| `correlation_penalty` | float | `0.3` | 相关性惩罚系数 |

**compute_weights 返回的 mode：**

| mode | 条件 | 行为 |
|------|------|------|
| `cold_start` | 历史 < 20 天 | 纯等权 1/N |
| `blending` | 20 ≤ 历史 < ic_lookback | 等权与 IC 加权的线性混合 |
| `ic_weighted` | 历史 ≥ ic_lookback | 纯 IC 加权 + 相关性惩罚 + 钳制 |

### EnsembleMonitor 融合质量监控

跟踪融合信号和各管线的贡献度变化，检测异常。

```python
from quantlab.signals.signal_ensemble import EnsembleMonitor

monitor = EnsembleMonitor(max_history=200)

# 每日记录
monitor.record("2024-06-28", weights, combined_signal, actual_return)

# 回填实际收益（次日补填）
monitor.update_actual_return("2024-06-27", next_day_return)

# 贡献度报告
report = monitor.get_contribution_report(lookback=60)
for name, stat in report.pipeline_stats.items():
    print(f"{name}: avg_weight={stat.avg_weight:.2f}, "
          f"marginal_ic={stat.marginal_ic:.3f}, "
          f"trend={stat.weight_trend}")

# 告警检查
alerts = monitor.check_anomaly()
for a in alerts:
    print(a)
```

**ContributionReport 结构：**

```python
ContributionReport
    period: str                           # "2024-04-01 ~ 2024-06-28"
    pipeline_stats: Dict[str, PipelineStat]

PipelineStat
    name: str                             # 管线名称
    avg_weight: float                     # 平均权重
    avg_ic: float                         # 平均 IC
    marginal_ic: float                    # 边际贡献 IC
    weight_trend: str                     # "increasing" | "stable" | "decreasing"
```

**4 类告警规则：**

| 编号 | 条件 | 告警消息 |
|------|------|---------|
| 1 | 融合信号 IC 连续 10 天 < 0 | `WARN: Combined signal IC negative for 10 consecutive days` |
| 2 | 某管线权重连续 20 天 = min_weight | `WARN: Pipeline {name} at minimum weight for 20 days, consider disabling` |
| 3 | 所有管线 IC 同时下降 | `WARN: All pipelines IC declining, possible regime change` |
| 4 | 融合信号被单管线独占（权重 > 0.95） | `WARN: Combined signal dominated by {name}, diversification lost` |

### SignalEnsemblePipeline 融合入口

日常回测/实盘入口，将三条管线信号合成为统一选股信号。

```python
from quantlab.signals.signal_ensemble import SignalEnsemblePipeline

ensemble = SignalEnsemblePipeline(
    ic_lookback=60,             # IC 窗口长度
    uncertainty_penalty=0.1,    # 不确定性惩罚系数
    min_weight=0.1,             # 单管线最低权重
    max_weight=0.6,             # 单管线最高权重
    correlation_penalty=0.3,    # 相关性惩罚系数
)

# ---- 融合 ----
signals = {
    "alpha":              alpha_signal,         # M2 输出（Series）
    "kronos":             kronos_return_1d,      # M3 输出（Series）
    "kronos_uncertainty": kronos_uncertainty,     # M3 不确定性（可选）
    "rdagent":            rdagent_signal,         # M4 输出（可为 None）
}
output = ensemble.combine(signals, "2024-06-28")

# ---- 使用输出 ----
output.signal              # Series: 融合信号（值域 (0, 1]）→ M6 选股
output.weights             # Dict: {"alpha": 0.45, "kronos": 0.33, "rdagent": 0.22}
output.mode                # "cold_start" / "blending" / "ic_weighted"
output.uncertainty_adjusted  # bool: 是否应用了不确定性惩罚
output.pipeline_ranks      # Dict[name, Series]: 各管线归一化后的 rank 信号

# ---- 历史更新 ----
# 次日收益已知后调用（用于 IC 计算）
ensemble.update_history("2024-06-28", signals, actual_return_0629)

# ---- 监控 ----
weights = ensemble.get_weights()
monitor = ensemble.get_monitor()
ic_df = ensemble.get_ic_history()
```

**构造参数：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `ic_lookback` | int | `60` | IC 窗口长度 |
| `uncertainty_penalty` | float | `0.1` | Kronos 不确定性惩罚系数 |
| `min_weight` | float | `0.1` | 单管线最低权重 |
| `max_weight` | float | `0.6` | 单管线最高权重 |
| `correlation_penalty` | float | `0.3` | 管线间相关性惩罚系数 |

**EnsembleOutput 结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `signal` | Series | 最终融合信号，值域 (0, 1]，高值看多 |
| `weights` | Dict[str, float] | 本次使用的各管线权重 |
| `pipeline_ranks` | Dict[str, Series] | 各管线归一化后的 rank 信号 |
| `uncertainty_adjusted` | bool | 是否应用了不确定性惩罚 |
| `mode` | str | `"cold_start"` / `"blending"` / `"ic_weighted"` |

## 架构与原理

### 融合流程

`combine()` 内部执行 9 步流程：

```
输入: signals = {alpha, kronos, kronos_uncertainty, rdagent}
                          │
  ┌───────────────────────┴───────────────────────┐
  │  Step 1: 过滤空管线                            │
  │  · rdagent=None → 移除                        │
  │  · 分离 kronos_uncertainty                     │
  └───────────────────────┬───────────────────────┘
                          │
  ┌───────────────────────┴───────────────────────┐
  │  Step 2: 预处理                               │
  │  · alpha:   winsorize(1%,99%) → rank_normalize │
  │  · kronos:  rank_normalize（不缩尾）           │
  │  · rdagent: winsorize(1%,99%) → rank_normalize │
  └───────────────────────┬───────────────────────┘
                          │
  ┌───────────────────────┴───────────────────────┐
  │  Step 3: 对齐股票 index                        │
  │  · 取全部管线的 index 并集                      │
  │  · 缺失值填 0.5（中性值）                       │
  └───────────────────────┬───────────────────────┘
                          │
  ┌───────────────────────┴───────────────────────┐
  │  Step 4: 计算权重                              │
  │  · ICWeightEngine.compute_weights()            │
  │  · 过滤掉不参与的管线 → 重新归一化              │
  └───────────────────────┬───────────────────────┘
                          │
  ┌───────────────────────┴───────────────────────┐
  │  Step 5: 加权融合                              │
  │  · raw_score = Σ(weight_i × rank_i)            │
  └───────────────────────┬───────────────────────┘
                          │
  ┌───────────────────────┴───────────────────────┐
  │  Step 6: 不确定性惩罚                          │
  │  · unc_rank = rank(kronos_uncertainty)         │
  │  · raw_score -= unc_rank × penalty (0.1)       │
  └───────────────────────┬───────────────────────┘
                          │
  ┌───────────────────────┴───────────────────────┐
  │  Step 7: 最终归一化                            │
  │  · signal = rank_normalize(raw_score)          │
  │  · 保证输出 ∈ (0, 1]                           │
  └───────────────────────┬───────────────────────┘
                          │
  ┌───────────────────────┴───────────────────────┐
  │  Step 8: 记录到 monitor                        │
  │  Step 9: 返回 EnsembleOutput                   │
  └───────────────────────────────────────────────┘
```

### IC 动态加权

权重分配基于各管线的**滚动 Rank IC**（Spearman 相关系数），而非固定权重：

```
IC_i = mean(spearmanr(signal_i_t, return_{t+1})) for t in recent ic_lookback days
```

权重计算流程：

```
1. 计算各管线滚动 IC
     ic_i = mean(daily_rank_ic) over ic_lookback days

2. 截断负 IC
     effective_ic_i = max(0, ic_i)

3. 归一化
     · 若全部 effective_ic = 0 → 等权
     · 否则 raw_weight_i = effective_ic_i / Σ effective_ic_j

4. 相关性感知调整（见下节）

5. 权重钳制
     · weight_i = clip(weight_i, min_weight, max_weight)
     · 重新归一化使 Σ weight = 1.0
```

### 冷启动与过渡

回测初期或系统上线初期，历史数据不足以可靠计算 IC，因此设计了三阶段过渡：

```
                   20天                    60天 (ic_lookback)
    ├─────────────────┤────────────────────────┤──────────→
    │   cold_start    │       blending         │  ic_weighted
    │   纯等权 1/N    │  α×IC权 + (1-α)×等权   │  纯 IC 加权
    │                 │  α = (天数-20)/(60-20) │
```

**为什么 20 天才开始过渡？**
- IC 估计需要一定样本量才有统计意义
- 20 天约 1 个月，是最短合理窗口
- 过早使用 IC 加权容易被噪声误导

**过渡期的混合公式：**
```
weight_i = α × ic_weight_i + (1 - α) × (1/N)
其中 α = (历史天数 - 20) / (ic_lookback - 20)，范围 [0, 1]
```

### 相关性感知降权

当两条管线的信号高度相关时，它们提供的信息是冗余的，应降低冗余管线的权重：

```
对每对管线 (A, B):
  计算历史平均截面 Spearman |corr|

  若 |corr| > 0.5:
    · 保留 IC 更高的管线权重不变
    · IC 较低的管线权重 × (1 - penalty × corr)
    · penalty 默认 0.3

  重新归一化
```

**示例：**
```
alpha IC=0.05, kronos IC=0.03, corr(alpha, kronos)=0.7

kronos 权重被惩罚:
  penalty = 1 - 0.3 × 0.7 = 0.79
  kronos_weight *= 0.79
  → alpha 相对权重上升
```

### 不确定性惩罚

Kronos 推理时对每只股票产出多采样预测，方差大表示不确定性高。不确定性惩罚是**个股级别**的，而非管线级别：

```
unc_rank = rank_normalize(kronos_uncertainty)     # 高不确定性 → 高 rank
penalty = unc_rank × uncertainty_penalty (0.1)
raw_score -= penalty
```

**为什么不直接降低 kronos 管线权重？**

- 权重是管线级别的，对所有股票生效
- Kronos 对某些股票预测置信度高、对另一些低
- 高置信度股票应保持 kronos 权重，低置信度应个股降权

**惩罚幅度 0.1 的含义：**

- uncertainty rank 最高的股票（最不确定）→ 得分下调 0.1
- uncertainty rank 最低的股票（最确定）→ 几乎不影响
- 0.1 约等于 rank 信号标准差的 1/3，是温和的调整

### 告警机制

EnsembleMonitor 持续记录融合结果，当检测到以下异常时产生告警：

| 告警 | 触发条件 | 含义 | 建议动作 |
|------|---------|------|---------|
| IC 连续负 | 融合 IC 连续 10 天 < 0 | 模型整体失效 | 检查市场异常、考虑降仓 |
| 权重最低值 | 某管线连续 20 天 = min_weight | 该管线无贡献 | 考虑禁用或触发 M4 再进化 |
| 全线下降 | 所有管线 IC 同时下降 | 可能发生 regime change | 降低仓位、增大等权比例 |
| 多样性丧失 | 融合信号被单管线独占（>0.95） | 退化为单管线 | 检查是否需要更多信号源 |

## 参数调优指南

### 核心参数选择

| 参数 | 保守 | 默认 | 激进 | 说明 |
|------|------|------|------|------|
| `ic_lookback` | 90 | 60 | 30 | 窗口越长 → 权重越稳定，但响应慢 |
| `uncertainty_penalty` | 0.05 | 0.1 | 0.2 | 越大 → 对不确定股票惩罚越重 |
| `min_weight` | 0.15 | 0.1 | 0.05 | 越大 → 弱管线保留更多权重 |
| `max_weight` | 0.5 | 0.6 | 0.8 | 越大 → 强管线可获得更高权重 |
| `correlation_penalty` | 0.1 | 0.3 | 0.5 | 越大 → 相关管线降权越狠 |

### 场景推荐

**稳健配置（推荐新手使用）：**

```python
ensemble = SignalEnsemblePipeline(
    ic_lookback=90,
    uncertainty_penalty=0.05,
    min_weight=0.15,
    max_weight=0.5,
)
```

权重切换慢，每条管线至少 15% 权重，单管线最多 50%。适合信号源质量不确定时。

**敏捷配置（市场变化快时使用）：**

```python
ensemble = SignalEnsemblePipeline(
    ic_lookback=30,
    uncertainty_penalty=0.15,
    min_weight=0.05,
    max_weight=0.8,
)
```

快速响应 IC 变化，允许强管线获得高权重。适合市场频繁切换风格时。

**双管线配置（无 RD-Agent 时）：**

```python
ensemble = SignalEnsemblePipeline(
    ic_lookback=60,
    uncertainty_penalty=0.1,
    min_weight=0.2,      # 两管线各至少 20%
    max_weight=0.8,
)
signals = {
    "alpha": alpha_signal,
    "kronos": kronos_signal,
    "kronos_uncertainty": kronos_unc,
    "rdagent": None,         # 跳过 M4
}
output = ensemble.combine(signals, date)
# weights: {"alpha": ~0.55, "kronos": ~0.45}
```

## 常见问题

### 1. 融合信号值域是什么？

输出 `signal` 值域为 **(0, 1]**，因为最终做了 rank percentile 归一化：
- **> 0.8**：强烈看多（Top 20%）
- **0.4 ~ 0.6**：中性
- **< 0.2**：强烈看空（Bottom 20%）

下游 M6 可直接对 `signal` 排序选出 Top-K 买入、Bottom-K 卖出。

### 2. 只有两条管线可以融合吗？

可以。将缺失管线设为 `None`：

```python
signals = {
    "alpha": alpha_signal,
    "kronos": kronos_signal,
    "rdagent": None,           # M4 跳过
}
output = ensemble.combine(signals, date)
# 自动用两管线融合，权重和 = 1.0
```

单管线也可融合（退化为该管线的 rank 信号）。

### 3. 冷启动阶段权重全一样，有意义吗？

有意义。冷启动阶段虽然权重等分，但融合仍然有效：
- **多样化收益**：不同管线捕捉不同市场信号
- **降低方差**：多信号平均可降低单管线的噪声
- **渐进过渡**：20 天后开始逐渐引入 IC 加权

### 4. update_history 必须调用吗？

如果需要 IC 动态加权（非等权），必须调用。否则永远处于 `cold_start` 模式。

调用时机：**次日收益已知后**，通常在 T+1 日开盘前调用：

```python
# T 日流程
output = ensemble.combine(signals_T, date_T)

# T+1 日流程（先回填 T 日收益）
ensemble.update_history(date_T, signals_T, actual_return_T1)
output = ensemble.combine(signals_T1, date_T1)
```

### 5. 权重为什么被钳制？

即使某管线 IC 暂时最高，权重也不超过 `max_weight`（默认 60%）：
- **IC 有噪声**：短窗口 IC 不代表真实信号质量
- **防过度集中**：某管线偶然表现好时不应独占权重
- **保留多样化**：多样化信号源是抵御 regime change 的最佳手段
- **min_weight 保留弱管线信息**：即使某管线暂时低迷，10% 权重保留其信息

### 6. 不确定性惩罚和降低 kronos 权重有什么区别？

| | 降低 kronos 权重 | 不确定性惩罚 |
|---|---|---|
| 粒度 | 管线级别（所有股票） | 个股级别 |
| 效果 | 所有股票的 kronos 贡献下降 | 只有不确定股票的得分下降 |
| 适用场景 | kronos 整体表现差 | kronos 部分预测不可靠 |

### 7. 融合信号 IC 为负怎么办？

收到告警 `Combined signal IC negative for 10 consecutive days` 时：

1. 检查各管线独立 IC（用 `ensemble_monitor.py ic-history`）
2. 如果所有管线 IC 都为负 → 可能是 regime change，考虑降仓
3. 如果只有某管线 IC 为负 → IC 加权会自动降其权重，等待自然修复
4. 可手动调大 `ic_lookback` 使权重更稳定

### 8. 如何知道某条管线有没有贡献？

使用 marginal IC（边际贡献 IC）：

```bash
python ensemble_monitor.py backtest \
    --alpha-signal signals_alpha.csv \
    --kronos-signal signals_kronos.csv \
    --return-csv returns.csv
```

如果某管线的 `marginal_ic ≈ 0`，说明它的信息已被其他管线覆盖。

### 9. 如何将 M5 输出传给 M6？

```python
from quantlab.signals.signal_ensemble import SignalEnsemblePipeline

ensemble = SignalEnsemblePipeline()
output = ensemble.combine(signals, date)

# output.signal 是 pd.Series(index=股票代码, values=融合得分)
# 直接传给 M6 的 OrderGenerator
order_gen.generate_buy_orders(output.signal, holdings, prices, ...)
order_gen.generate_sell_orders(output.signal, holdings, prices, ...)
```

### 10. 回测时每次都要重建 ensemble 吗？

不需要。`SignalEnsemblePipeline` 是有状态的，内部维护 IC 历史和 monitor 记录。在回测循环中复用同一个实例：

```python
ensemble = SignalEnsemblePipeline(ic_lookback=60)

for date in trading_dates:
    # 获取各管线信号
    signals = get_signals(date)

    # 融合
    output = ensemble.combine(signals, date)

    # 用融合信号做交易决策
    execute_trades(output.signal, date)

    # 次日收益回填（下一轮循环开始前）
    if prev_signals:
        ensemble.update_history(prev_date, prev_signals, get_return(date))

    prev_date, prev_signals = date, signals
```
