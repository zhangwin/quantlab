## 模块简介

M6 交易执行是 QuantLab 的执行层，负责将 M5 融合信号转化为实际交易：

1. **订单生成** — 根据融合信号 + 当前持仓决定买卖标的、数量和目标价
2. **成交判定** — 以 T+1 开盘价模拟真实成交，处理涨跌停、价格偏离、资金不足
3. **账户管理** — 维护现金、持仓、交易记录，确保资金守恒

所有脚本位于 `quantlab/execution/` 目录下。

**在系统中的位置：**

```
M5 信号融合 ──→ 融合信号
                    ↓
              M6 交易执行
              ├── T 日: 生成买卖订单
              │   ├── 信号驱动（买/卖）
              │   └── 风控驱动（止损/行业/熔断）← M7 风控
              └── T+1 日: 成交判定
                  ├── 涨跌停检查
                  ├── 价格偏离检查
                  ├── 滑点 + 佣金 + 印花税
                  └── 账户更新
                    ↓
              持仓 + 交易记录 ──→ M8 归因分析
                               ──→ M9 主循环
```

**M6 的核心设计原则：**

| 原则 | 说明 |
|------|------|
| A 股真实约束 | 佣金万 2.5、印花税万 5、T+1、涨跌停、100 股整手 |
| 卖出优先 | 先卖后买，卖出回款当日可用于买入 |
| 风控优先 | 熔断(1) > 止损(2) > 行业减仓(3) > 信号卖(4) > 信号买(5) |
| 资金守恒 | 任何时刻 cash + 持仓市值 + 累计费用 = 初始资金 ± 盈亏 |
| 优雅降级 | 资金不足时部分成交，涨跌停时记录失败原因 |

## 环境准备

M6 依赖极少，M1 环境已装好即可：

```bash
pip install numpy pandas   # 通常已安装
```

无需 GPU、无需 Qlib 数据。M6 本身不访问原始行情数据库，只接收：
- M5 融合信号（pd.Series 或 CSV）
- 行情价格数据（收盘价、开盘价、涨跌停价）

## 快速开始

### 单日交易执行

```bash
cd quantlab/execution

# 假设 M5 已输出融合信号 CSV
# signal.csv: 行=日期, 列=股票, 值=融合信号得分
# close.csv / open.csv: 行=日期, 列=股票, 值=价格

# 第一步：执行单日交易（T 日信号 → T+1 成交）
python execution_run.py --anchor-date 2024-06-28 \
    --signal ./outputs/ensemble/signal.csv \
    --close-prices ./data/close.csv \
    --open-prices ./data/open.csv

# 第二步：查看成交明细
python execution_status.py history --trades ./outputs/execution/trades.csv

# 第三步：查看持仓
python execution_status.py holdings --trades ./outputs/execution/trades.csv \
    --close-prices ./data/close.csv --date 2024-07-01
```

### 多日回测

```bash
# 连续执行多个交易日
python execution_run.py --start 2024-01-01 --end 2024-06-30 \
    --signal ./outputs/ensemble/signals.csv \
    --close-prices ./data/close.csv \
    --open-prices ./data/open.csv \
    --limit-up ./data/limit_up.csv \
    --limit-down ./data/limit_down.csv \
    --output ./outputs/execution/trades.csv \
    --summary-output ./outputs/execution/daily_summary.csv

# 查看回测摘要
python execution_status.py summary \
    --summary ./outputs/execution/daily_summary.csv

# 查看交易成本
python execution_status.py cost --trades ./outputs/execution/trades.csv

# 查看成功率
python execution_status.py stats --trades ./outputs/execution/trades.csv
```

### Python API 执行

```python
from quantlab.execution.execution import ExecutionPipeline

pipeline = ExecutionPipeline(
    initial_cash=1_000_000,
    max_positions=20,
    max_single_weight=0.1,
    target_buy_count=3,
    target_sell_count=3,
)

# T 日收盘后: 生成订单
orders = pipeline.generate_orders(signal, close_prices, industry_map, "2024-06-28")

# 追加风控强卖（M7 传入）
orders = pipeline.add_force_sell_orders(orders, ["SH600000"], close_prices, "stop_loss")

# T+1 日开盘: 执行
records = pipeline.execute_orders(orders, open_prices, limit_up, limit_down, "2024-07-01")

# 查看当日摘要
summary = pipeline.get_daily_summary(close_prices_t1, "2024-07-01")

```

## 脚本详细说明

### execution_run.py 交易执行

日常交易执行和回测的**主入口**，从融合信号生成订单并模拟成交。

**单日执行：**

```bash
# 基本用法
python execution_run.py --anchor-date 2024-06-28 \
    --signal ./outputs/ensemble/signal.csv \
    --close-prices ./data/close.csv \
    --open-prices ./data/open.csv

# 带涨跌停约束
python execution_run.py --anchor-date 2024-06-28 \
    --signal ./outputs/ensemble/signal.csv \
    --close-prices ./data/close.csv \
    --open-prices ./data/open.csv \
    --limit-up ./data/limit_up.csv \
    --limit-down ./data/limit_down.csv

# 带行业映射
python execution_run.py --anchor-date 2024-06-28 \
    --signal ./outputs/ensemble/signal.csv \
    --close-prices ./data/close.csv \
    --open-prices ./data/open.csv \
    --industry-map ./data/industry.csv
```

**多日回测：**

```bash
python execution_run.py --start 2024-01-01 --end 2024-06-30 \
    --signal ./outputs/ensemble/signals.csv \
    --close-prices ./data/close.csv \
    --open-prices ./data/open.csv \
    --output ./outputs/execution/trades.csv \
    --summary-output ./outputs/execution/daily_summary.csv
```

**自定义执行参数：**

```bash
python execution_run.py --anchor-date 2024-06-28 \
    --signal ./outputs/ensemble/signal.csv \
    --close-prices ./data/close.csv \
    --open-prices ./data/open.csv \
    --initial-cash 2000000 \
    --max-positions 30 \
    --max-single-weight 0.08 \
```

**追加止损强卖：**

```bash
python execution_run.py --anchor-date 2024-06-28 \
    --signal ./outputs/ensemble/signal.csv \
    --close-prices ./data/close.csv \
    --open-prices ./data/open.csv \

```

**完整参数列表：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--signal` | 必填 | 融合信号 CSV |
| `--close-prices` | 必填 | 收盘价 CSV (date×symbol) |
| `--open-prices` | 必填 | 开盘价 CSV (date×symbol) |
| `--limit-up` | 无 | 涨停价 CSV |
| `--limit-down` | 无 | 跌停价 CSV |
| `--industry-map` | 无 | 行业映射 CSV (symbol,industry) |
| `--anchor-date` | 无 | T 日日期（单日模式） |
| `--start` / `--end` | 无 | 日期范围（多日模式） |
| `--initial-cash` | 1,000,000 | 初始资金 |
| `--max-positions` | 20 | 最大持仓数 |
| `--max-single-weight` | 0.1 | 单票仓位上限（占总资产比例） |
| `--target-buy` | 3 | 每日目标买入数 |
| `--target-sell` | 3 | 每日目标卖出数 |
| `--force-sell` | 无 | 强制卖出股票代码（逗号分隔） |
| `--force-sell-reason` | stop_loss | 强卖原因: stop_loss / industry_limit / circuit_breaker |
| `--output` | 无 | 输出成交记录 CSV |
| `--summary-output` | 无 | 输出每日摘要 CSV |
| `--quiet` | False | 静默模式 |

### PortfolioAccount 账户管理

管理现金、持仓和交易记录。

```python
from quantlab.execution.execution import PortfolioAccount

acct = PortfolioAccount(initial_cash=1_000_000)
```

**方法：**

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `get_cash()` | — | float | 当前可用现金 |
| `get_holdings()` | — | Dict[str, Position] | 当前持仓快照 |
| `get_portfolio_value(prices)` | 当前价格 dict | float | 现金 + 持仓市值 |
| `get_market_value(prices)` | 当前价格 dict | float | 纯持仓市值 |
| `get_holding_weight(symbol, prices)` | 股票, 价格 | float | 该股票占总资产比例 |
| `update_highest_price(prices)` | 当前价格 dict | — | 更新持仓最高价（止损用） |
| `apply_buy(symbol, shares, price, cost, date)` | 成交信息 | — | 买入: 扣现金、更新持仓 |
| `apply_sell(symbol, shares, price, proceeds, date)` | 成交信息 | — | 卖出: 加现金、减持仓 |
| `can_sell(symbol, exec_date)` | 股票, 日期 | bool | T+1 限制检查 |
| `get_trade_history()` | — | List[TradeRecord] | 全部交易记录 |

**apply_buy 加仓逻辑：**

```python
# 已持有时: 加权平均更新成本价
acct.apply_buy("SH600000", 500, 10.0, 5005.0, "2024-06-01")
acct.apply_buy("SH600000", 500, 12.0, 6005.0, "2024-06-02")
pos = acct.get_holdings()["SH600000"]
# pos.shares = 1000, pos.cost_price ≈ 11.0
```

### OrderGenerator 订单生成

根据融合信号和当前持仓生成买卖订单。

```python
from quantlab.execution.execution import OrderGenerator

gen = OrderGenerator(
    max_positions=20,       # 最大持仓数
    max_single_weight=0.1,  # 单票仓位上限
    target_buy_count=3,     # 每日目标买入数
    target_sell_count=3,    # 每日目标卖出数
)
```

**方法：**

| 方法 | 说明 |
|------|------|
| `generate_sell_orders(signal, holdings, prices, exec_date, account)` | 信号驱动的卖出订单 |
| `generate_buy_orders(signal, holdings, prices, industry_map, total_value, available_cash)` | 信号驱动的买入订单 |
| `generate_force_sell_orders(symbols, prices, holdings, reason)` | 风控强卖订单 |
| `generate_liquidation_orders(holdings, prices)` | 熔断全部清仓订单 |

**使用示例：**

```python
signal = pd.Series({"A": 0.9, "B": 0.7, "C": 0.3, "D": 0.1})

# 卖出: 信号低于中位数的持仓
sell_orders = gen.generate_sell_orders(
    signal, account.get_holdings(), close_prices, "2024-06-28", account,
)

# 买入: 信号最高的非持仓股
buy_orders = gen.generate_buy_orders(
    signal, account.get_holdings(), close_prices,
    industry_map, total_value=1_000_000, available_cash=200_000,
)

# 风控强卖
stop_orders = gen.generate_force_sell_orders(
    ["A", "B"], close_prices, account.get_holdings(), "stop_loss",
)
```

### OrderExecutor 成交判定

以 T+1 开盘价模拟真实成交，处理所有 A 股约束。

```python
from quantlab.execution.execution import OrderExecutor

executor = OrderExecutor()
records = executor.execute(
    orders,                  # 待执行订单
    open_prices,             # T+1 开盘价
    limit_up,                # 涨停价
    limit_down,              # 跌停价
    "2024-07-01",            # 执行日期
    account,                 # 账户
)
```

**成交状态枚举：**

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| `filled` | 全额成交 | 正常成交 |
| `partial_filled` | 部分成交 | 资金仅够买部分（100 股整数倍） |
| `failed_limit_up` | 涨停买不到 | 开盘价 ≥ 涨停价 |
| `failed_limit_down` | 跌停卖不掉 | 开盘价 ≤ 跌停价 |
| `failed_price_deviation` | 价格偏离 | 买入: 开盘价 > 目标价 |
| `failed_no_cash` | 资金不足 | 连 100 股都买不起 |
| `failed_t1_restriction` | T+1 限制 | 当日买入的股票同日卖出 |

## 架构与原理

### 执行流程

```
T 日收盘后                              T+1 日开盘
    │                                       │
    ▼                                       ▼
┌─────────────┐                     ┌──────────────┐
│ M5 融合信号  │                     │ T+1 开盘价    │
└──────┬──────┘                     │ 涨停价/跌停价  │
       │                            └──────┬───────┘
       ▼                                   │
┌──────────────┐                           │
│ OrderGenerator│                           │
│  ├ sell_orders│  ←── 信号 < 中位数        │
│  ├ buy_orders │  ←── 信号最高 + 约束      │
│  └ force_sell │  ←── M7 风控             │
└──────┬───────┘                           │
       │                                   │
       ▼                                   ▼
┌────────────────────────────────────────────┐
│             OrderExecutor                   │
│  按 priority 排序:                          │
│    1. circuit_breaker → 2. stop_loss →      │
│    3. industry_limit → 4. signal_sell →     │
│    5. signal_buy                            │
│                                             │
│  逐个判定:                                   │
│    卖出: T+1检查 → 跌停检查 → 成交(含滑点)    │
│    买入: 涨停检查 → 价格偏离 → 资金检查 → 成交 │
└──────────────┬──────────────────────────────┘
               │
               ▼
┌──────────────────────┐
│   PortfolioAccount    │
│  ├ 扣/加现金          │
│  ├ 更新持仓            │
│  └ 记录 TradeRecord   │
└──────────────────────┘
```

### A 股交易约束

M6 完整模拟 A 股真实交易环境，所有约束均体现在执行逻辑中：

**成本项：**

| 项目 | 费率 | 适用方向 | 计算方式 |
|------|------|----------|----------|
| 佣金 | 万 2.5 (0.025%) | 买卖双向 | `max(amount × 0.00025, 5)` |
| 印花税 | 万 5 (0.05%) | 仅卖出 | `amount × 0.0005` |
| 滑点 | 千 1 (0.1%) | 买卖双向 | 买: `open × 1.001`，卖: `open × 0.999` |

**成本示例：**

```
买入 10 元 × 1000 股:
  exec_price = 10 × 1.001 = 10.01
  amount     = 10.01 × 1000 = 10,010
  commission = max(10010 × 0.00025, 5) = 5.00（触发最低佣金）
  stamp_tax  = 0（买入不收）
  总成本     = 10,010 + 5 = 10,015

卖出 10 元 × 10000 股:
  exec_price = 10 × 0.999 = 9.99
  amount     = 9.99 × 10000 = 99,900
  commission = max(99900 × 0.00025, 5) = 24.975
  stamp_tax  = 99900 × 0.0005 = 49.95
  净回款     = 99,900 - 24.975 - 49.95 = 99,825.075
```

**交易限制：**

| 限制 | 检查逻辑 | 失败状态 |
|------|----------|----------|
| T+1 | `last_buy_date < exec_date` | `failed_t1_restriction` |
| 涨停板 | `open_price ≥ limit_up` | `failed_limit_up` |
| 跌停板 | `open_price ≤ limit_down` | `failed_limit_down` |
| 100 股整手 | `floor(shares / 100) × 100` | 不足 100 股则跳过 |
| 资金不足 | 尝试减少到可买手数 | `partial_filled` 或 `failed_no_cash` |

---

### 订单优先级机制

所有订单按 priority 值升序执行（低值优先）：

```
Priority 1: circuit_breaker  熔断清仓 → 全部持仓立即卖出
Priority 2: stop_loss        止损卖出 → 回撤达到阈值的个股
Priority 3: industry_limit   行业减仓 → 超出行业集中度的个股
Priority 4: signal_sell      信号卖出 → 信号衰弱的持仓
Priority 5: signal_buy       信号买入 → 信号最强的新标的
```

**设计要点：**

- 卖出一定先于买入执行，释放的现金可立即用于买入
- 风控订单不会被资金不足阻塞（因为它们先执行）
- 同一股票有多个卖出订单时，`add_force_sell_orders` 自动去重，保留更高优先级

---

### 卖出候选判定

`generate_sell_orders` 使用两个条件筛选卖出候选：

```
条件 1（信号衰弱）:
  持仓股票的 signal_score < 全市场中位数
  → 说明信号已不在前 50%，应该让位给更优标的

条件 2（长期低迷）:
  持仓天数 > 60 天 AND signal_score < 全市场 60 分位
  → 长期持有但信号持续不强，可能是价值陷阱

满足任一条件 → 加入候选
按 signal_score 升序排列（最差优先卖）
取前 target_sell_count 只
全仓卖出（不做部分减仓）
target_price = 收盘价 × 0.95（跌停附近放弃）
```

---

### 买入约束体系

`generate_buy_orders` 层层筛选：

```
Step 1: 候选池
  ├ 融合信号按得分降序
  ├ 排除已持仓
  └ 排除当日卖出的票

Step 2: 行业约束
  ├ max_industry_count = max(1, max_positions // 5)
  └ 某行业已达上限 → 跳过该行业新票

Step 3: 仓位约束
  └ 单票资金 = min(available_cash / target_buy_count,
                    total_value × max_single_weight)

Step 4: 手数约束
  ├ target_price = 收盘价 × 1.02（允许 2% 溢价）
  ├ shares = floor(单票资金 / target_price / 100) × 100
  └ shares < 100 → 跳过

Step 5: 取前 target_buy_count 只
```

---

### 成交判定逻辑

`OrderExecutor` 对每个订单按以下流程判定：

**卖出订单：**

```
① T+1 检查 → 当日买入不可卖 → failed_t1_restriction
② 跌停检查 → 开盘价 ≤ 跌停价 → failed_limit_down
③ 无论价格是否低于 target_price → 仍然成交（优先清仓）
④ 成交:
   exec_price = open_price × (1 - 0.001)
   amount = exec_price × shares
   commission = max(amount × 0.00025, 5)
   stamp_tax = amount × 0.0005
   proceeds = amount - commission - stamp_tax
   → account.apply_sell(...)
```

**买入订单：**

```
① 涨停检查 → 开盘价 ≥ 涨停价 → failed_limit_up
② 价格偏离 → 开盘价 > 目标价 → failed_price_deviation
③ 资金检查:
   exec_price = open_price × (1 + 0.001)
   needed = exec_price × shares + max(exec_price × shares × 0.00025, 5)
   if needed > cash:
     尝试减少: shares = floor(cash / exec_price / 100) × 100
     shares < 100 → failed_no_cash
     否则 → partial_filled
④ 成交:
   amount = exec_price × shares
   commission = max(amount × 0.00025, 5)
   cost = amount + commission
   → account.apply_buy(...)
```

---

### 资金流转

```
初始状态:
  cash = 1,000,000
  holdings = {}

T 日生成订单:
  ┌──────────────────────────────────────────┐
  │ 预估卖出回款:                              │
  │   est_proceeds = Σ(卖出市值 - 预估费用)     │
  │ available_cash = cash + est_proceeds      │
  │   → 用于计算买入订单的资金分配               │
  └──────────────────────────────────────────┘

T+1 执行（按 priority 顺序）:
  卖出 A:  cash += proceeds_A      ← 回款立即可用
  卖出 B:  cash += proceeds_B
  买入 C:  cash -= cost_C          ← 使用卖出回款
  买入 D:  cash -= cost_D

任何时刻:
  cash + Σ(shares × price) = initial_cash ± Σ(realized_pnl) - Σ(fees)
```

## 参数调优指南

### 持仓管理参数

| 参数 | 保守 | 均衡（默认） | 激进 |
|------|------|--------------|------|
| `max_positions` | 30 | 20 | 10 |
| `max_single_weight` | 0.05 | 0.10 | 0.20 |
| `target_buy_count` | 2 | 3 | 5 |
| `target_sell_count` | 2 | 3 | 5 |

**参数含义与影响：**

- `max_positions`：越大越分散，降低个股风险，但降低集中度收益。建议：大盘 ≥ 20，小盘 ≤ 10
- `max_single_weight`：单票最大仓位占比。0.05 = 最多 5% 在一只股票上。越小越分散
- `target_buy_count`：每日最多新买入几只。越大换仓越快，交易成本越高
- `target_sell_count`：每日最多卖出几只。越大换仓越快

### 资金规模建议

| 初始资金 | max_positions | max_single_weight | 说明 |
|----------|---------------|-------------------|------|
| 50 万 | 10 | 0.15 | 小资金集中持有 |
| 100 万 | 20 | 0.10 | 标准配置 |
| 500 万 | 30 | 0.05 | 大资金充分分散 |
| 1000 万+ | 50 | 0.03 | 机构级分散 |

### 换仓频率与成本的权衡

```
假设: 100 万资金，单票 5 万

每日买 3 + 卖 3:
  日均成交 ≈ 30 万
  日均费用 ≈ 30 万 × (0.025% + 0.025% + 0.05%/2) = 45 元
  年均费用 ≈ 45 × 244 ≈ 10,980 元 ≈ 1.1% 资产

每日买 5 + 卖 5:
  日均成交 ≈ 50 万
  年均费用 ≈ 18,300 元 ≈ 1.8% 资产

结论: 高频换仓需确保信号 alpha 能覆盖交易成本
```

## 常见问题

### 1. 为什么卖出时信号低于 target_price 仍然成交？

这是有意为之。卖出的目的是清仓离场，`target_price = 收盘价 × 0.95` 只是参考价。只有跌停（`open_price ≤ limit_down`）才会真正阻止卖出。风控卖出更是无条件执行（除非跌停）。

### 2. 为什么买入时 open_price > target_price 就不买？

买入时我们设定了 2% 的溢价上限（`target_price = close × 1.02`）。如果 T+1 开盘价跳涨超过 2%，说明这只股票的买入成本超出预期，放弃可以避免追涨。

### 3. T+1 回款当日可用吗？

是的。M6 的执行顺序保证卖出先于买入，卖出的回款（`apply_sell` 后 cash 增加）在同一批次执行中即可用于买入。

### 4. 部分成交是什么意思？

当账户现金不足以买入目标股数时，OrderExecutor 会自动计算最大可买手数（100 股整数倍）。如果至少能买 100 股，则以 `partial_filled` 状态成交；如果连 100 股都买不起，则 `failed_no_cash`。

### 5. 加仓时成本价怎么算？

加权平均法：

```
原持仓: 500 股 × 10 元 = 5000
加仓:   500 股 × 12 元 = 6000
新均价: (5000 + 6000) / (500 + 500) = 11 元
```

注意：`cost_price` 是执行价格的加权平均，不含佣金。佣金在 `apply_buy` 的 cost 参数中体现，直接从 cash 扣除。

### 6. 如何模拟熔断？

```python
orders = pipeline.generate_orders(signal, close_prices, None, T)
orders = pipeline.add_liquidation_orders(orders, close_prices)
# → 清除所有买入订单，所有持仓生成 priority=1 的卖出订单
records = pipeline.execute_orders(orders, open_prices, limit_up, limit_down, T_next)
```

### 7. generate_orders 的卖出回款预估准确吗？

预估使用 `收盘价 × (1 - 滑点) - 预估费用`，与实际成交会有差异（开盘价 ≠ 收盘价）。这个预估只用于给买入订单分配资金，不影响实际成交。实际执行时以真实开盘价为准。

### 8. 如何与 M7 风控配合？

M7 检查止损、行业暴露等，输出需要强卖的股票列表和原因：

```python
# M7 输出
stop_loss_list = risk.check_stop_loss(holdings, prices)
industry_reduce = risk.check_industry_exposure(holdings, prices)

# M6 执行
orders = pipeline.generate_orders(signal, prices, industry_map, T)
orders = pipeline.add_force_sell_orders(orders, stop_loss_list, prices, "stop_loss")
orders = pipeline.add_force_sell_orders(orders, industry_reduce, prices, "industry_limit")

# 如果触发熔断
if risk.check_circuit_breaker(nav_series):
    orders = pipeline.add_liquidation_orders(orders, prices)

records = pipeline.execute_orders(orders, open_prices, limit_up, limit_down, T_next)
```

### 9. 涨跌停价数据从哪来？

A 股涨跌停价 = 前收盘价 × (1 ± 10%)（创业板/科创板 20%）。如果不提供涨跌停 CSV（`--limit-up` / `--limit-down`），则不做涨跌停检查（所有订单都可成交）。回测时建议务必提供。

### 10. 如何查看某日为什么没买某只股票？

可能的原因：
- 已持有（排除持仓股）
- 行业已满（行业约束）
- 资金不够一手（手数约束）
- 涨停（成交判定失败）
- 开盘价超出目标价（价格偏离）
- 排名不够靠前（只买 top target_buy_count 只）

查看交易记录中的 failed 状态可定位具体原因：

```bash
python execution_status.py history \
    --trades ./outputs/execution/trades.csv \
    --start 2024-06-28 --end 2024-06-28
```
